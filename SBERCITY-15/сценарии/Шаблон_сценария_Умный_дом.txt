
1. Шаблон сценария: Поддержка Умный дом

1.a. Умный дом
1.a.i. Квартира

Датчик движения
Климат система
Освещение

1.2 Триггеры
Умный дом как система в целом:
• Не работает умный дом
• Система не отвечает
• Не подключается приложение
• Ошибка системы
• Не работает управление
• Система зависла
• Нет связи с устройствами

Датчик движения:
• Не работает датчик движения
• Датчик не реагирует
• Ложные срабатывания
• Свет не включается
• Свет включается сам
• Проблемы с датчиком

Климат система:
• Не работает климат система
• Не регулируется температура
• Не работает кондиционирование
• Не включается вентиляция
• Не работает теплый пол
• Проблемы с климатом

Освещение:
• Не работает освещение
• Не включается свет
• Свет мигает
• Не работает сценарий освещения
• Не реагирует управление светом
• Освещение включается само

1.3 Термины
• Умный дом — система автоматизированного управления устройствами квартиры.
• Датчик движения — устройство фиксации движения для автоматизации сценариев.
• Климат система — система управления температурой, вентиляцией и климатом.
• Сценарий освещения — автоматическое или дистанционное управление освещением.
• Приложение умного дома — мобильное приложение для управления системой.

2. Развилка
2.1 Когда это консультация
• Житель спрашивает, как работает система умного дома.
• Житель уточняет возможности датчиков или сценариев.
• Житель интересуется настройками климат системы.
• Житель спрашивает, как управлять освещением.
• Житель уточняет порядок подключения устройств.

2.2 Когда это заявка
• Житель сообщает о неисправности системы.
• Житель сообщает о неработающем датчике.
• Житель сообщает о проблеме с климат системой.
• Житель сообщает о неисправности освещения.
• Житель просит провести диагностику.

3. Консультация
3.1 Ответ
Если у вас возникли проблемы с системой умного дома, датчиками, климат системой или освещением, мы можем оформить заявку на диагностику и проверку оборудования.
Перед оформлением заявки желательно уточнить, какое именно устройство или функция работают некорректно.

3.2 Уточнения
Базовые:
• Подскажите, пожалуйста, какая именно проблема возникла?
• Какое устройство работает некорректно?
• Когда появилась проблема?
• Проблема постоянная или периодическая?
• Приложение отображает ошибку?

Для системы умного дома:
• Не работает вся система или отдельные устройства?
• Есть ли подключение к интернету?
• Приложение запускается?

Для датчика движения:
• Датчик не реагирует или срабатывает некорректно?
• Проблема наблюдается постоянно?
• Есть ли механические повреждения?

Для климат системы:
• Не регулируется температура?
• Не работает вентиляция или кондиционирование?
• Проблема наблюдается во всех помещениях?

Для освещения:
• Не работает отдельная зона или весь свет?
• Свет не включается вручную или через приложение?
• Есть ли мерцание или ложные включения?

4. Заявка
4.1 Обязательные атрибуты
4.1.1 Для заявки: Поддержка → Умный дом → Квартира → Умный дом как система в целом
Основные атрибуты:
• Описание проблемы;
• Какие функции не работают;
• Наличие ошибок в приложении;
• Когда появилась проблема;
• Фото/видео ошибки при наличии;
• Дата и удобный интервал для связи.

4.1.2 Для заявки: Поддержка → Умный дом → Квартира → Датчик движения
Основные атрибуты:
• Тип проблемы;
• Где расположен датчик;
• Как проявляется неисправность;
• Когда появилась проблема;
• Фото/видео при наличии;
• Дата и удобный интервал для связи.

4.1.3 Для заявки: Поддержка → Умный дом → Квартира → Климат система
Основные атрибуты:
• Какая функция не работает;
• Температура или режим работы;
• Где наблюдается проблема;
• Когда появилась проблема;
• Фото/видео при наличии;
• Дата и удобный интервал для связи.

4.1.4 Для заявки: Поддержка → Умный дом → Квартира → Освещение
Основные атрибуты:
• Какая зона освещения не работает;
• Как проявляется неисправность;
• Работает ли управление через приложение;
• Когда появилась проблема;
• Фото/видео при наличии;
• Дата и удобный интервал для связи.

В случае отсутствия верификации пользователя необходимо дополнительно запросить:
• адрес: дом, квартира;
• ФИО;
• номер телефона.

Данные, которые необходимо получить из CRM УК при верификации:
• адрес помещения;
• ФИО жителя/собственника;
• номер телефона;
• наличие уже открытых заявок по системе умного дома.

4.2 Логика диалога для создания заявки
• Определить тип обращения: консультация или заявка.
• Уточнить тип неисправности:
- система умного дома;
- датчик движения;
- климат система;
- освещение.
• Собрать обязательные атрибуты.
• Проверить наличие массовых сбоев.
• Предложить приложить фото/видео.
• Создать заявку после достаточности данных.

4.3 Проверки
• Проверить наличие открытых заявок по тому же адресу.
• Проверить достаточность данных:
- тип устройства;
- описание проблемы;
- время возникновения;
- контактные данные.
• Фото/видео не является обязательным.

5. Исключения
• Проблема связана с отсутствием интернета — сообщить о необходимости проверки подключения.
• Повреждение оборудования по вине пользователя — перевести на диспетчера.
• Не удалось определить тип неисправности — передать оператору.
• Пользователь требует срочного выезда вне регламента — передать диспетчеру.

6. Эскалация
• Полный отказ системы умного дома.
• Повторяющиеся сбои после ранее выполненного ремонта.
• Подозрение на короткое замыкание или запах гари.
• Неисправность влияет на безопасность жителя.
• Не удалось собрать обязательные атрибуты.

7. Шаблоны ответа клиенту
7.1 Если консультация
«Я могу помочь оформить заявку по системе умного дома. Подскажите, пожалуйста, какое устройство или функция работают некорректно?»

7.2 Если заявка
Общий старт:
«Конечно, помогу оформить заявку. Подскажите, пожалуйста, какая именно проблема наблюдается?»

Для датчика движения:
«Подскажите, пожалуйста, как именно работает датчик: не реагирует, срабатывает сам или работает нестабильно?»

Для климат системы:
«Подскажите, пожалуйста, какая именно функция климат системы работает некорректно?»

Для освещения:
«Подскажите, пожалуйста, какая зона освещения не работает и как проявляется проблема?»

Фото/видео:
«Если у вас есть фото или видео проблемы, пожалуйста, приложите материалы. Если сейчас неудобно, заявку можно оформить без вложений.»

7.3 Когда заявка создана
«Заявка по системе умного дома создана. Могу ли я еще чем-нибудь помочь?»
