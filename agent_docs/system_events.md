# System Events (message_type 200)

When `message.message_type === 200`, the message is a system event generated by the platform rather than a user message. These events notify the agent of dialog lifecycle changes, operator assignments, scoring, and other platform-level actions.

---

## Event-handler agents

Some agents are designed exclusively as event handlers — they **only ever receive `message_type === 200`** and never receive user messages.

Such agents are registered in the platform settings as system event subscribers, not assigned to dialogs directly. They are not invoked by the user and cannot be called via `/redirect` or `/switch`. The platform dispatches events to them automatically whenever a matching event occurs.

**Implications for implementation:**

- No need to gate on `message_type` — every invocation is guaranteed to be `200`.
- The script can skip the `message_type === 1` branch entirely.
- The entrypoint can go straight to reading `domain_event_type`.

```js
// Event-handler agent — receives only message_type === 200
async function run() {
    const event = getEventType(message)
    logger.info(`EventHandler: received "${event}"`)

    switch (event) {
        case "DialogFinished":
            // ...
            break
        default:
            logger.info(`EventHandler: unhandled event "${event}", skipping`)
    }

    return [agentApi.makeAckReply()]
}
```

Contrast with a **mixed agent** that handles both user messages and system events — see the Full Skeleton section below.

---

## How to Detect the Event Type

The event name is stored as a slot value inside `message.slot_context.filled_slots`. The slot with `slot_id === "domain_event_type"` holds the event name string.

**Canonical helper pattern (recommended):**

```js
const EVENT_SLOT_ID = "domain_event_type"

function getEventType(msg) {
    return msg.slot_context?.filled_slots?.find(
        slot => slot.slot_id === EVENT_SLOT_ID
    )?.value
}
```

**Dispatch by event type:**

```js
async function run() {
    if (message.message_type === 200) {
        const event = getEventType(message)
        switch (event) {
            case "DialogCreated":
                // ...
                break
            case "DialogFinished":
                // ...
                break
            default:
                logger.info(`SystemEvent: unhandled event "${event}", skipping`)
                return [agentApi.makeAckReply()]
        }
    }

    if (message.message_type === 1) {
        // handle normal user message
    }

    return [agentApi.makeAckReply()]
}
```

## Event Reference

| Event | Description |
|---|---|
| `DialogCreated` | A new dialog was opened |
| `DialogFinished` | The dialog was closed/finished |
| `DialogAddedToQueue` | Dialog entered an operator queue (on creation or when an operator returned it to queue) |
| `DialogDetached` | Dialog was detached from an operator |
| `DialogScoreReceived` | The client submitted a CSAT score |
| `IncomingMessageProcessed` | An incoming channel message was fully processed by the platform |
| `UserFilledSlotsChanged` | Filled slots on the user were updated |
| `UserPhoneChanged` | The client's phone number was changed |
| `TaskRoutingFound` | The dialog was assigned to an operator |
| `SupervisorConnectedToDialog` | A supervisor joined the dialog |
| `SupervisorDisconnectedFromDialog` | A supervisor left the dialog |

### Additional events found in the codebase

| Event | Description |
|---|---|
| `ScoreRequest` | A CSAT score request was sent to the client |
| `ReadConfirmation` | A message read confirmation was received |

## Sub-field: DialogFinishReason

When handling `DialogFinished`, an additional slot `DialogFinishReason` may be present with a string describing the reason for closing:

```js
case "DialogFinished": {
    const reason = message.slot_context?.filled_slots?.find(
        slot => slot.slot_id === "DialogFinishReason"
    )?.value
    // known values: "timeout", "user_merge"
    logger.info(`Dialog finished, reason: ${reason}`)
    break
}
```

## Ancillary Fields Inside Event Handlers

Beyond the event type, scripts commonly read these fields when processing system events:

| Field | Typical use |
|---|---|
| `message.routing.operator_id` | Which operator was assigned (`TaskRoutingFound`, `DialogFinished`) |
| `message.user.omni_user_id` | Scope key for `omniUserStorage` |
| `message.user.session_id` | Dialog session ID |
| `message.channel.channel_id` | Gate on a specific channel before acting |
| `message.channel.customer_id` | `ProjectId` for `agentApi.sendMessage` calls |
| `message.context` | Array of `[key, value]` pairs — e.g. filtered for `queue_id` |

Slots commonly carried with system events:

| Slot ID | Description |
|---|---|
| `ep_DialogId` | Dialog ID |
| `ep_Score` | CSAT score value |
| `ep_Operator` | Operator display name |
| `ep_OperatorId` | Operator ID |
| `sys_phone` | Client phone number |
| `sys_omniuserid` | OmniUser ID |

## Full Skeleton

A complete agent that handles both system events and normal messages:

```js
const EVENT_SLOT_ID = "domain_event_type"

function getEventType(msg) {
    return msg.slot_context?.filled_slots?.find(
        slot => slot.slot_id === EVENT_SLOT_ID
    )?.value
}

function getSlotValue(slotId, msg) {
    return msg.slot_context?.filled_slots?.find(
        slot => slot.slot_id === slotId
    )?.value
}

async function run() {
    // --- System events ---
    if (message.message_type === 200) {
        const event = getEventType(message)
        logger.info(`MyAgent: system event "${event}"`)

        switch (event) {
            case "DialogCreated": {
                // e.g. initialise per-dialog storage
                await agentStorage.dialogStorage.set("started_at", Date.now())
                break
            }
            case "DialogFinished": {
                const reason = getSlotValue("DialogFinishReason", message)
                logger.info(`MyAgent: dialog finished, reason="${reason}"`)
                // e.g. send stats to external system
                break
            }
            case "TaskRoutingFound": {
                const operatorId = message.routing?.operator_id
                logger.info(`MyAgent: assigned to operator ${operatorId}`)
                break
            }
            case "DialogScoreReceived": {
                const score = getSlotValue("ep_Score", message)
                logger.info(`MyAgent: CSAT score received: ${score}`)
                break
            }
            default:
                logger.info(`MyAgent: unhandled event "${event}", skipping`)
        }

        return [agentApi.makeAckReply()]
    }

    // --- Normal user message ---
    if (message.message_type === 1) {
        const text = message.message.text
        logger.info(`MyAgent: user message: "${text}"`)
        return [agentApi.makeTextReply("Got your message!")]
    }

    return [agentApi.makeAckReply()]
}

run()
    .then(res => resolve(res))
    .catch(er => {
        logger.error(`MyAgent: ${er}`)
        resolve([agentApi.makeAckReply()])
    })
```
